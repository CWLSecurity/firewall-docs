# Firewall Vault — Audit Status (MVP)

## Current status
Firewall Vault is currently in MVP stage.

## Audit statement
At this stage, the product should be treated as:

- not audited, unless explicitly stated otherwise
- experimental MVP software
- suitable for careful early users and controlled rollout
- requiring independent user verification before serious use

## What this means in practice
Users should assume:

- smart contract risk may still exist
- UI and integration risk may still exist
- security assumptions may evolve
- documentation may improve as the product matures

## Recommended user posture
Before using Firewall Vault with meaningful funds:

- verify contract addresses
- verify source code on explorer
- understand current limitations
- test with low-value usage first
- treat MVP status seriously

## Future direction
A later version of Firewall Vault should include:
- formal audit process
- clearer audit/public review history
- stronger trust and verification materials
- production hardening beyond MVP

## Summary
Firewall Vault aims to improve security through on-chain enforcement, but MVP status and current audit status must be understood clearly by every early user.
