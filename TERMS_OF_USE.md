# Firewall Vault — Terms of Use (Pilot)

Last updated: 2026-04-22

## 1) Stage and scope
Firewall Vault is currently operated in pilot stage.

The active product scope is:
- `firewall-wallet` (on-chain contracts),
- `firewall-ui` (security console),
- queue automation bot runtime.

## 2) User responsibility
Users remain responsible for:
- custody of signer wallet keys/seed phrases,
- device and browser security,
- validating wallet addresses and transactions before signing.

## 3) No absolute guarantees
Firewall Vault provides deterministic on-chain policy enforcement.

It does not guarantee prevention of all losses or all attack classes.

## 4) Third-party dependencies
Service behavior depends on external infrastructure:
- wallet extensions,
- RPC providers,
- blockchain network conditions.

## 5) Pilot-risk acknowledgement
Unless explicitly announced otherwise, pilot users should treat the system as evolving software and start with low-value flows.
